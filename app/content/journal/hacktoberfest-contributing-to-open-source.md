# Hacktoberfest – Contributing to Open Source

Hi there! Hope everyone is doing well and taking care.

We all know that open source is a boon to the world. It has helped countless people **learn, create, and excel**. Many developers have overcome the fear of sharing their work thanks to amazing open source communities and contributions.

There's a famous saying:

> Sharing is caring.

Once you start contributing to open source, there's no going back.

## What is Hacktoberfest?

🎃 **Hacktoberfest** is an annual open source festival organized by DigitalOcean and community partners.

It's an incredible initiative for budding developers and designers. The event motivates people to make the world a better place by fixing bugs, improving documentation, and contributing new ideas to open source projects.

One of the best parts about open source is that if you encounter a bug in an application, you can actually fix it yourself and submit a Pull Request (PR). That's the beauty of open source.

## My Experience

I remember participating in **Major League Hacking (MLH)** where our team was using a library that powered a chat window.

We discovered a bug where a specific image format wasn't supported during uploads.

Instead of working around it, we:

1. Investigated the issue
2. Fixed the bug
3. Raised a Pull Request

And that's exactly how many open source journeys begin.

## How to Start Your Hacktoberfest Journey

Getting started is surprisingly simple.

### 1. Find Hacktoberfest Repositories

Browse repositories participating in Hacktoberfest:

https://github.com/topics/hacktoberfest

### 2. Choose Your Preferred Language

For example, if you're interested in JavaScript:

https://github.com/topics/hacktoberfest?l=javascript

### 3. Look for Beginner-Friendly Issues

Search for:

```text
is:open is:issue label:"good first issue" label:hacktoberfest
```

Or use:

https://github.com/issues?q=is%3Aopen+is%3Aissue+label%3A%22good+first+issue%22+label%3A%22hacktoberfest%22

### 4. Filter by Technology

Once you've found repositories, narrow them down by language and technology stack to match your skills.

### 5. Read the Documentation

Before contributing:

* Read the README
* Read CONTRIBUTING.md
* Understand the project structure

### 6. Start Small

If you're a beginner, choose simple issues first.

Building confidence is more important than tackling complex features.

### 7. Get Assigned

Whenever possible, ask maintainers to assign the issue to you.

This helps avoid duplicate work.

### 8. Create a Proper Pull Request

After fixing the issue:

* Raise a Pull Request
* Mention the issue number
* Link the issue in the PR description

This helps GitHub automatically connect the PR and issue.

### 9. Complete Four Valid Contributions

Traditionally, participants needed:

* 4 valid Pull Requests
* Accepted by maintainers
* Marked as Hacktoberfest eligible

Always check the current year's official rules.

### 10. Submit Before the Deadline

Make sure your contributions are reviewed and accepted before the event ends.

## Final Thoughts

Open source is one of the fastest ways to:

* Improve your coding skills
* Learn real-world development
* Collaborate with experienced developers
* Build your portfolio
* Give back to the community

Whether you're a beginner or an experienced developer, Hacktoberfest is a great opportunity to start contributing.

Happy Coding! 🚀
